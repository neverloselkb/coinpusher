# 🎰 [치즈 코인 푸셔] 기술 요구사항 명세서 (Tech Spec)

## 📌 프로젝트 개요
치지직(CHZZK) 스트리머를 위한 실시간 인터랙티브 3D 웹 미니게임 오버레이. 시청자의 채팅과 후원(치즈) 데이터를 물리 엔진 기반의 '동전 밀기' 게임으로 시각화하여 방송의 몰입도와 수익성을 높임.

---

## 1. 프론트엔드 (화면 & 물리 엔진)
* **3D 그래픽 렌더링:** `Three.js` 또는 `React Three Fiber (R3F)`
  * 투명한 유리관, 네온사인 이펙트, 금속 재질의 푸셔 텍스처(질감) 구현.
* **물리 엔진 (Physics):** `Cannon.js` 또는 `Rapier.js` (최근 R3F와 궁합이 좋은 최신 물리 엔진)
  * 동전의 무게, 중력, 마찰력을 계산하여 자연스럽게 쏟아지는 물리 법칙 적용.
* **UI/UX 프레임워크:** `React.js` 또는 `Vue.js`
  * 상태 관리(Zustand, Redux 등)를 통해 내기 진행 바, 피버 게이지, 실시간 팝업 등을 오버레이로 띄움.

## 2. 백엔드 & API 연동 (데이터 처리)
* **실시간 통신:** `Node.js` + `Socket.io` (또는 `WebSocket`)
  * 시청자가 채팅을 치거나 후원을 하는 즉시 웹 화면으로 데이터를 전송.
* **치지직 데이터 연동:**
  * **채팅 데이터:** 치지직 채팅 서버(WebSocket) 연결을 통해 실시간 메시지 파싱.
  * **후원 데이터:** 치즈 후원 알림 데이터 추출 후 클라이언트로 이벤트 브로드캐스팅.

## 3. 스트리머 웹 제어판 (Admin 페이지)
* **인증 및 DB:** `Firebase` (Auth, Firestore) 또는 `Supabase`
  * 스트리머 로그인, 고유 설정값(내기 목표, 푸셔 속도, 오버레이 URL) 저장 및 실시간 동기화.
* **배포 및 호스팅:** `Vercel` 또는 `Netlify` (프론트엔드), `Render` 또는 `Fly.io` (백엔드)

## 4. 방송 송출 호환성
* **OBS / XSplit 브라우저 소스 최적화:**
  * CSS `body { background: transparent; overflow: hidden; }` 적용.
  * 해상도: 1920x1080 (FHD) 16:9 픽스.

## 📚 5. 참고 자료 및 추천 오픈소스 (References & Resources)

### 💡 3D 및 물리 엔진 학습 자료
* **Three.js Journey (Bruno Simon):** 3D 웹 개발을 배울 수 있는 전 세계에서 가장 유명하고 완벽한 튜토리얼 (유료지만 가치 100%).
* **React Three Fiber 공식 문서 (docs.pmnd.rs):** React 환경에서 3D와 물리 엔진(Rapier)을 쉽게 세팅하는 예제 코드들이 가득함.
* **CodeSandbox 예제 검색:** 'React Three Fiber Physics' 또는 'Coin Pusher'를 검색하면 다른 개발자들이 만들어둔 유사한 물리 엔진 데모 코드를 참고할 수 있음.

### 🔌 치지직(CHZZK) API 연동 참고
* **치지직 비공식 API 라이브러리 (npm `chzzk`):** 국내 개발자들이 치지직 채팅과 후원 데이터를 쉽게 가져올 수 있도록 만들어둔 오픈소스 라이브러리 (GitHub에서 'chzzk api nodejs' 검색 시 다수 존재).
* **치지직 개발자 센터 (공식):** 공식적으로 제공되는 API 가이드라인 확인 (플랫폼 정책 변경에 대비).

### 🛠️ 기타 개발 팁
* **OBS 브라우저 소스 개발자 도구 활용:** 개발 중인 웹을 OBS에 띄워놓고 우클릭 > '상호작용' 또는 '원격 디버깅'을 통해 방송 화면 상에서의 성능(프레임 드랍 등)을 실시간으로 체크할 것.
